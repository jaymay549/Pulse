# AI-Synthesized Vendor Theme Summaries

## Date: 2026-02-17

## Problem

The "What Dealers Appreciate" and "Common Concerns" sections on vendor profiles are weak. They `GROUP BY title` on `vendor_mentions` and show the top 5 raw mention titles with a random quote — essentially a repackaged version of the mentions grid below. No synthesis, no intelligence, nothing worth paywalling.

These sections also have no pro-gating — they're visible to all users despite being the highest-value content on the page.

## Solution

Replace the raw title grouping with AI-synthesized theme summaries, generated by Gemini Flash and cached per vendor. Gate the content behind pro membership using the existing blur pattern.

## Data Model

New table:

```sql
vendor_theme_summaries (
  vendor_name                  text PRIMARY KEY,
  themes                       jsonb NOT NULL,
  mention_count_at_generation  int,
  generated_at                 timestamptz DEFAULT now()
)
```

The `themes` JSONB structure:

```json
{
  "positiveThemes": [
    {
      "label": "Hands-on onboarding",
      "summary": "Multiple dealers highlight dedicated implementation reps who stay involved through go-live. Several noted the vendor flew staff on-site for training.",
      "mention_count": 12
    }
  ],
  "warningThemes": [
    {
      "label": "Slow support response",
      "summary": "Dealers report 24-48 hour wait times on support tickets, with several noting escalation is required to get resolution on critical issues.",
      "mention_count": 8
    }
  ]
}
```

3-4 themes per sentiment. Each has a crisp label, a synthesized summary, and a mention count.

## Generation — Edge Function

A Supabase edge function `generate-vendor-themes` that:

1. Takes a vendor name (or "all" to batch-regenerate)
2. Fetches all `vendor_mentions` for that vendor, grouped by sentiment
3. Sends to Gemini Flash with a structured prompt
4. Upserts the result into `vendor_theme_summaries`

Prompt approach:

```
You are analyzing dealer feedback about {vendor_name}, an automotive industry vendor.

Below are {count} dealer mentions, grouped by sentiment.

POSITIVE MENTIONS:
- "{title}" — "{quote}"
- ...

WARNING MENTIONS:
- "{title}" — "{quote}"
- ...

Synthesize 3-4 themes per sentiment. For each theme:
- label: 2-5 word theme name (e.g., "Hands-on onboarding", "Pricing transparency")
- summary: 1-2 sentences synthesizing what dealers are saying. Reference patterns across multiple mentions, not single anecdotes.
- mention_count: how many of the provided mentions relate to this theme

Return JSON. Do not invent information not present in the mentions.
```

**Regeneration triggers:**
- Manual: admin can call the edge function per-vendor or for all
- Automatic: the function checks `mention_count_at_generation` vs current count — skips vendors with no new mentions
- Start simple: manual or weekly cron. Don't over-engineer the trigger.

**Minimum threshold:** Only generate for vendors with 5+ mentions. Below that, fall back to current `GROUP BY title` approach or show "Not enough data for AI summary."

**Cost:** Gemini Flash at ~$0.075/1M input tokens. A vendor with 50 mentions is ~2K tokens. 500 vendors = ~1M tokens = ~$0.075 per full regeneration.

## Pro-Gating

Non-pro users see:
- Section headers ("What Dealers Appreciate" / "Common Concerns") — visible
- Theme content — blurred with `blur-[6px]` and `select-none`, with a gradient fade overlay
- A compact upgrade CTA overlaid on the blurred content: lock icon + "Upgrade to unlock dealer insights"
- Clicking the CTA triggers `setShowUpgradeModal(true)` (authenticated) or opens Stripe checkout (unauthenticated)

Pro users see full synthesized themes, no blur.

Implementation: a conditional wrapper around the theme lists using the existing `isProUserValue` check. Same blur pattern as `VendorCard.tsx`.

## Frontend Changes

**VendorProfile.tsx** theme sections updated to render AI summaries:

Per theme item:
- **Label** in semibold (e.g., "Hands-on onboarding")
- **Mention count** badge (same emerald/red pill as today)
- **Summary** — the 1-2 sentence synthesis replaces the raw `sample_quote`

**`get_vendor_themes` RPC** rewritten to read from `vendor_theme_summaries` table instead of doing `GROUP BY title`. Same return interface — just better content behind it.

**Fallbacks:**
- Vendors with <5 mentions: show current `GROUP BY title` output with "Based on limited data" label
- Vendors with no cached summary yet: show current approach until edge function runs

## What Changes

| Component | Change |
|-----------|--------|
| `vendor_theme_summaries` table | New |
| `generate-vendor-themes` edge function | New |
| `get_vendor_themes` RPC | Rewrite to read cached summaries |
| `VendorProfile.tsx` theme sections | Display summaries + pro-gating blur |

## What Doesn't Change

- `vendor_mentions` table (source data unchanged)
- Other vendor profile sections (stats, compared vendors, mentions grid)
- `VendorCard` component
- Search, trending, category pages
- AI extraction pipeline
