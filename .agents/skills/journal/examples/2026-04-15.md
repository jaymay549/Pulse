### Stripe webhook race condition — diagnosed, fix pending soak

**Branch:** `fix/webhook-idempotency` · **By:** Jordan · **Commits:** `m1n2o3p`

Support flagged three double-charge tickets this week. Jordan traced it to our webhook handler processing `payment_intent.succeeded` twice when Stripe retried on a slow response — the idempotency check was happening *after* the DB insert, so the second delivery created a duplicate charge row.

Fix moves the idempotency check to the top of the handler using the `event.id` as the key. Deployed to staging; waiting 24h before prod to make sure no regressions show up in the retry path.

**Impact:** Stops double-charges; the 0.3% affected customers get an automated refund script running tomorrow.

### Auth middleware refactor — merged

**Branch:** `refactor/auth-middleware` · **By:** Sam · **Commits:** `q1r2s3t`, `u4v5w6x`

Three-week-long branch closed today. The old middleware stored session tokens in a way legal flagged as non-compliant; Sam moved token storage into an HttpOnly cookie with a rotating CSRF pair and deleted the old localStorage path.

Main complication was a Safari ITP edge case where the CSRF cookie was being dropped on cross-subdomain requests. Fix was to set `SameSite=Lax` and move the API to the same apex domain.

**Impact:** Unblocks the compliance review scheduled for end of month.
