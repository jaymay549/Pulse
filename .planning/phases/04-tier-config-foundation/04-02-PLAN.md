**1. Create `src/hooks/useTierConfig.ts`:**

A React Query hook that provides tier config data and mutation:

```typescript
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { useClerkSupabase } from "@/hooks/useClerkSupabase";
import type { TierComponentConfig, VendorTier, ComponentVisibility } from "@/types/tier-config";
```

- **`useTierConfig()`** returns `{ configs, isLoading, error, updateVisibility }`:
  - `configs`: `TierComponentConfig[]` from `useQuery` calling `supabase.rpc("get_tier_component_config")` with queryKey `["tier-component-config"]`.
  - `updateVisibility`: `useMutation` calling `supabase.rpc("upsert_tier_component_config", { p_tier, p_component_key, p_visibility })`.
    - `onMutate`: Optimistic update — update the config in the query cache immediately.
    - `onError`: Roll back optimistic update, show `toast.error("Failed to save")`.
    - `onSuccess`: Show `toast.success("Saved")`, invalidate `["tier-component-config"]`.
  - The RPC calls use `supabase.rpc(name, params)` pattern — since types.ts is auto-generated and may not have the RPC typed, cast as needed: `(supabase.rpc as any)("get_tier_component_config")`.

- Export a helper: `getVisibility(configs, tier, componentKey)` that looks up the visibility for a given tier+component from the configs array. Returns `'full'` as default if not found.

**2. Create `src/pages/admin/TierConfigPage.tsx`:**

Follow the existing admin page pattern (see VendorManagementPage for reference):

- Import `useTierConfig` hook, `DASHBOARD_COMPONENTS`, `TIER_LABELS`, `VendorTier`, `ComponentVisibility` from `@/types/tier-config`.
- Import shadcn components: `Select, SelectContent, SelectItem, SelectTrigger, SelectValue` from `@/components/ui/select`, `Badge` from `@/components/ui/badge`, `Loader2` from `lucide-react`.

**Page layout:**
- Page title: "Tier Component Configuration" with subtitle "Control which dashboard sections each vendor tier can access"
- A grid/table with:
  - **Rows**: Grouped by `group` from DASHBOARD_COMPONENTS (3 groups: "Analytics & Insights", "Engagement & Activity", "Presence & Catalog"). Each group has a header row with the group name.
  - **Columns**: Component label | Tier 1 | Tier 2 | Test
  - **Cells**: Each tier column cell is a `<Select>` dropdown with options: "Full" (value: full), "Gated" (value: gated), "Hidden" (value: hidden).
  - The current value comes from `getVisibility(configs, tier, componentKey)`.
  - On value change, call `updateVisibility.mutate({ p_tier: tier, p_component_key: componentKey, p_visibility: newValue })`.

**Visual design (Tailwind, matching admin dark theme):**
- Use a `<table>` with `border-collapse` inside a rounded card (`bg-zinc-900 border border-zinc-800 rounded-xl overflow-hidden`).
- Group header rows: `bg-zinc-800/50 text-xs font-bold text-zinc-400 uppercase tracking-wider`.
- Component label cells: `text-sm text-zinc-200 font-medium`.
- Select dropdowns: Style to match admin dark theme.
- **Test tier column**: Add a subtle dashed left border (`border-l border-dashed border-zinc-700`) to visually separate it from production tiers, per ACONF-03.
- Color-code the Select trigger text:
  - `full` = green (`text-emerald-400`)
  - `gated` = amber (`text-amber-400`)
  - `hidden` = red (`text-red-400`)

**Loading state**: Show `<Loader2>` spinner centered while configs load.
**Error state**: Show error message with retry button.

Export as default: `export default TierConfigPage`.
