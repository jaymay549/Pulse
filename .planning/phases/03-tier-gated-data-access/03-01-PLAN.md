<objective>
Database-level tier enforcement: create vendor_tier() and auth_vendor_name() SECURITY DEFINER helpers, enable RLS on vendor_mentions with tier-gated policies, and add auth guards to T2-only RPCs.

Purpose: This is the security foundation of the entire phase. Without DB-level enforcement, frontend gating alone can be bypassed via direct API/RPC calls. These migrations make it impossible for a T1 vendor to access T2 data or for Vendor A to see Vendor B's data, regardless of how the frontend is configured.

Output: Three migration files applied to Supabase that enforce vendor isolation and tier boundaries at the database level.
</objective>

    **Migration 1: `supabase/migrations/20260413200000_vendor_tier_function.sql`**

    Create two SECURITY DEFINER helper functions for RLS policies:

    1. `public.vendor_tier()` — Returns the tier string ('unverified', 'tier_1', 'tier_2') for the current vendor session, or NULL for Clerk/anon sessions.

    ```sql
    CREATE OR REPLACE FUNCTION public.vendor_tier()
    RETURNS TEXT
    LANGUAGE sql
    STABLE
    SECURITY DEFINER
    SET search_path = public
    AS $$
      SELECT tier
      FROM public.vendor_logins
      WHERE user_id = auth.uid()
      LIMIT 1;
    $$;

    REVOKE ALL ON FUNCTION public.vendor_tier() FROM PUBLIC;
    GRANT EXECUTE ON FUNCTION public.vendor_tier() TO authenticated;
    ```

    2. `public.auth_vendor_name()` — Returns the vendor_name for the current vendor session, or NULL for Clerk/anon sessions.

    ```sql
    CREATE OR REPLACE FUNCTION public.auth_vendor_name()
    RETURNS TEXT
    LANGUAGE sql
    STABLE
    SECURITY DEFINER
    SET search_path = public
    AS $$
      SELECT vendor_name
      FROM public.vendor_logins
      WHERE user_id = auth.uid()
      LIMIT 1;
    $$;

    REVOKE ALL ON FUNCTION public.auth_vendor_name() FROM PUBLIC;
    GRANT EXECUTE ON FUNCTION public.auth_vendor_name() TO authenticated;
    ```

    Both functions use `STABLE` (not `IMMUTABLE`) because they read from a table. Both use `SET search_path = public` to prevent search_path injection. Both REVOKE from PUBLIC and GRANT to authenticated only.

    **Migration 2: `supabase/migrations/20260413300000_vendor_mentions_rls.sql`**

    Enable RLS on `vendor_mentions` (currently has NO RLS — verified in research). CRITICAL: Must add permissive read policies for anon and Clerk-authenticated users in the SAME migration to avoid breaking the existing dealer-facing vendor feed (/vendors, /vendors/:slug).

    ```sql
    -- Enable RLS (currently off — table predates migration-managed schema)
    ALTER TABLE public.vendor_mentions ENABLE ROW LEVEL SECURITY;

    -- Preserve existing anon read behavior (dealer public feed)
    CREATE POLICY "Public read vendor_mentions"
      ON public.vendor_mentions FOR SELECT
      TO anon
      USING (true);

    -- Authenticated users: Clerk sessions pass through, vendor sessions T2-only + own vendor
    CREATE POLICY "Authenticated read vendor_mentions"
      ON public.vendor_mentions FOR SELECT
      TO authenticated
      USING (
        -- Clerk sessions (vendor_tier() returns NULL): preserve existing public read
        public.vendor_tier() IS NULL
        OR
        -- Vendor sessions: T2 only, own vendor data only (case-insensitive)
        (
          public.vendor_tier() = 'tier_2'
          AND lower(vendor_name) = lower(public.auth_vendor_name())
        )
      );
    ```

    IMPORTANT: The `vendor_tier() IS NULL` branch is essential — without it, Clerk-authenticated dealer users would get zero rows because `NULL = 'tier_2'` evaluates to NULL (denied) in SQL. See Research Pitfall 2.

    Use `lower()` on both sides of the vendor_name comparison to handle casing mismatches between vendor_logins (exact from vendor_profiles) and vendor_mentions (normalized via canonical_vendor_name_case). See Research Pitfall 4.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vendor JWT -> RLS | Vendor's Supabase Auth JWT carries auth.uid() used by vendor_tier() and auth_vendor_name() |
| Vendor -> SECURITY DEFINER RPCs | Vendor can call any RPC with any p_vendor_name parameter |
| Clerk/anon -> vendor tables | Existing dealer/admin queries must not be broken by new RLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | vendor_mentions | mitigate | RLS policy: vendor_tier() = 'tier_2' AND lower(vendor_name) = lower(auth_vendor_name()) |
| T-03-02 | Elevation of Privilege | get_vendor_dimensions RPC | mitigate | Auth guard: vendor_tier() <> 'tier_2' -> return empty |
| T-03-03 | Elevation of Privilege | get_vendor_dashboard_intel RPC | mitigate | Strip 'recommendations' key for non-T2 vendor sessions |
| T-03-04 | Information Disclosure | All vendor RPCs | mitigate | Vendor isolation: lower(p_vendor_name) <> lower(auth_vendor_name()) -> return empty |
| T-03-05 | Denial of Service | vendor_mentions RLS | mitigate | anon + Clerk read policies in same migration as ENABLE RLS to prevent feed breakage |
| T-03-06 | Information Disclosure | vendor_tier() / auth_vendor_name() | mitigate | REVOKE ALL FROM PUBLIC, GRANT TO authenticated only; SECURITY DEFINER + SET search_path |
| T-03-07 | Spoofing | vendor_tier() | accept | Function trusts auth.uid() from Supabase Auth JWT; JWT signing is Supabase's responsibility |
</threat_model>
