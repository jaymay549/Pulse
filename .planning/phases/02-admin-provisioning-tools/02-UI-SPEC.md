| Dominant (60%) | `bg-zinc-950` | Page background, sidebar background |

## Interaction Contracts

### Wizard State Machine

```
Step 1 (Email)
  → validation: non-empty, valid email format
  → "Next" disabled until valid email entered
  → on Next: advance to Step 2

Step 2 (Link Profile)
  → validation: non-empty, must match a value from the datalist
  → "Next" disabled until a profile is selected from the autocomplete
  → on Next: advance to Step 3

Step 3 (Set Tier)
  → validation: one of [unverified, tier_1, tier_2] must be selected
  → "Next" disabled until tier selected (default: no pre-selection)
  → on Next: advance to Step 4

Step 4 (Confirm)
  → "Provision Vendor" triggers mutation
  → mutation loading: button disabled + spinner
  → on success: close dialog, reset all state, show success toast, invalidate query
  → on error: show error toast, stay on Step 4 (do not reset state)

Dialog close (any step):
  → reset: step → 1, email → "", vendorName → "", tier → ""
  → Source: Pitfall 5 from RESEARCH.md
```

### Resend Invite

- One "Resend Invite" button per table row in the Actions column
- Click triggers mutation immediately (no confirmation dialog — resend is non-destructive)
- Loading state: button shows `<Loader2 className="animate-spin h-3 w-3" />` inline, `disabled`
- No resend cooldown in Phase 2 (deferred to v2 SEC-02)
- Success/error feedback via Sonner toast

### Table Filtering

- Single search input (`<Input>` with `<Search>` icon prefix) filtering by vendor name or email — client-side filter on fetched data
- No server-side pagination in Phase 2 (vendor account count is small; full list fetched once)
- No column sort in Phase 2 (default order: newest first, from RPC `ORDER BY created_at DESC`)
