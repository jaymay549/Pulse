## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Tampering | Vendor name free-text bypass | mitigate | VendorWizardDialog validates `vendorProfiles.includes(vendorName)` before enabling Next on step 2 -- prevents typos that would break Phase 3 RLS joins |
| T-02-09 | Information Disclosure | Vendor email list in admin table | accept | Data is business vendor emails visible only to admin-authenticated users; not PII |
| T-02-10 | Spoofing | Wizard CSRF | accept | All mutations use Clerk JWT auth token; standard SPA pattern; no CSRF tokens needed for API calls with bearer auth |
| T-02-11 | Denial of Service | Rapid wizard submissions | accept | Admin-only surface; rate limiting deferred to v2 SEC-02 |
</threat_model>
