| Token | Value | Usage |

## Color

Source: `src/index.css` CSS custom properties ‚Äî full token set established.

| Role | Token | HSL Value | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | Page background, login card background |
| Secondary (30%) | `--muted` | `hsl(214 20% 96%)` | OTP step background tint, form container subtle fill |
| Accent (10%) | `--primary` | `hsl(214 100% 50%)` | Reserved for specific elements only (see below) |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | Error messages, invalid OTP state, expired session alert |

Accent (`--primary`, blue hsl 214 100% 50%) reserved for:
1. "Send Code" button (primary CTA on email step)
2. "Verify Code" button (primary CTA on OTP step)
3. "Vendor Login" nav link in `Navigation.tsx`
4. Focus ring on all interactive inputs (via `--ring`)
5. `InputOTP` slot active/focused border

Accent is NOT used for: body text, card backgrounds, dividers, error states, loading states.

Dark mode: All tokens defined in `.dark` class in `index.css`. Phase 1 components inherit automatically via Tailwind `dark:` prefix classes ‚Äî no new dark mode tokens needed.

## Layout Specification

Primary visual anchor: the full-width "Send Code" / "Verify Code" CTA button, which occupies the bottom of the card and draws the eye to the single available action at each step.

### VendorLoginPage

```
min-h-screen
  flex items-center justify-center
    bg-background
      p-4

  [Card ‚Äî max-w-md w-full]
    CardHeader
      CardTitle (h2, 20px black)   "Vendor Portal"
      CardDescription (14px 400)   Context line (see Copywriting)
    CardContent
      [Step 1 ‚Äî Email]
        Label                       "Email address"
        Input type="email"          placeholder="you@company.com"
        Button full-width default   "Send Code"
      [Step 2 ‚Äî OTP]  (replaces Step 1 content, same card)
        p text 14px muted           Confirmation line (see Copywriting)
        InputOTP maxLength={6}      centered, 44px slot height
        Button full-width default   "Verify Code"
        Button ghost sm             "Request a new code"
        Button ghost sm             "‚Üê Use a different email"
```

Single-card two-step flow: step 1 and step 2 render inside the same `Card` ‚Äî step 2 replaces step 1 content in-place (no page navigation, no modal).

### VendorAuthGuard loading state

```
min-h-screen
  flex items-center justify-center
    bg-background
      Loader2 h-8 w-8 animate-spin text-muted-foreground
```

## Interaction States

### Email Input (Step 1)

| State | Visual |
|-------|--------|
| Default | Standard `Input` component with border `--input` |
| Focus | Blue ring via `--ring` (built into shadcn Input) |
| Loading (OTP sending) | Input disabled, button shows `Loader2` spinner + "Sending..." label |
| Error (unknown email) | Red text below input using `text-destructive text-sm` |
| Error (rate limited) | Same treatment as unknown email; different copy |

### OTP Slots (Step 2)

| State | Visual |
|-------|--------|
| Default | `InputOTPSlot` default border |
| Active (current slot) | Blue border via `--ring` (built into InputOTP) |
| Filled | No special treatment ‚Äî slot shows digit |
| Error (wrong/expired code) | Red border on all slots: add `border-destructive` class to `InputOTPGroup`; error text below |
| Loading (verifying) | Button shows `Loader2` spinner + "Verifying..." label; slots disabled |

### CTA Buttons

| State | Visual |
|-------|--------|
| Default | `Button variant="default"` ‚Äî blue background |
| Hover | `bg-primary/90` (built into variant) |
| Disabled / Loading | `disabled:opacity-50` (built into Button) + `Loader2` spinner left of label |
| Focus-visible | Blue ring via `--ring` (built into Button) |

## Accessibility Constraints

- OTP slots must have `aria-label="Digit {n} of 6"` ‚Äî handled by `InputOTPSlot` component internally.
- Email `Input` must have an associated `Label` via `htmlFor` / `id` pair.
- Error messages must use `role="alert"` so screen readers announce them without focus shift.
- `Loader2` spinner inside buttons must have `aria-hidden="true"` and button must update its visible label to "Sending..." / "Verifying...".
- Minimum touch target for all interactive elements: 44px (OTP slots: set `h-11` on `InputOTPSlot`).
