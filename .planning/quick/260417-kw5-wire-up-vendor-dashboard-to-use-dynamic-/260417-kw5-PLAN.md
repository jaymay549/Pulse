**In src/hooks/useTierConfig.ts:**

Add a new exported hook `useTierConfigReadonly()` below the existing `useTierConfig()`.
This hook fetches tier config using the default anon Supabase client (`import { supabase } from "@/integrations/supabase/client"`) instead of `useClerkSupabase()`. This is necessary because vendor-auth sessions (magic link via vendorSupabase) do not have Clerk tokens, but the `tier_component_config` table has no RLS so the anon client works fine.

```typescript
export function useTierConfigReadonly() {
  const { data: configs = [], isLoading, error } = useQuery<TierComponentConfig[]>({
    queryKey: QUERY_KEY,
    queryFn: async () => {
      const { data, error } = await supabase.rpc("get_tier_component_config" as any);
      if (error) throw error;
      return data as TierComponentConfig[];
    },
    staleTime: 5 * 60 * 1000, // 5 min — config rarely changes
  });
  return { configs, isLoading, error };
}
```

Note: Reuse the same `QUERY_KEY` so admin pages and vendor dashboard share cache when both are loaded. Add `staleTime: 5 * 60 * 1000` since config changes rarely.
